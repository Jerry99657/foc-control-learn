# JerryFOC - STM32G4 轻量级高性能无刷电机 FOC 驱动库

![License](https://img.shields.io/badge/License-MIT-blue.svg)
![Platform](https://img.shields.io/badge/Platform-STM32G431-green.svg)
![Language](https://img.shields.io/badge/Language-C-orange.svg)

JerryFOC 是一个从零手写、专为 **STM32G4 系列**（基于 STM32G431CBT6）深度优化的轻量级 FOC (磁场定向控制) 驱动库。它摒弃了臃肿的冗余抽象，直接利用 STM32G4 强大的底层硬件特性（如硬件三角函数加速器、TIM-ADC 底层硬件触发同步等），实现了极致的控制频率与极低的 CPU 占用。

## ✨ 核心特性

- **硬件级波谷电流采样**：配置 TIM1 CH4 生成精准的 TRGO 触发信号，让 ADC 完美踩在 PWM 波谷进行相电流采样，彻底免疫开关噪声（实现真正的“纯硬件单边底层对齐采样”）。
- **MT6701 磁编码器支持**：通过 SPI DMA 零 CPU 开销获取 14-bit 高精度绝对位置。
- **专为云台电机优化的“电压闭环模式”**：针对内阻大、电感极小（如 2804 电机）的云台电机，绕过易受噪声干扰的高频电流环，使用极其丝滑安静的纯电压前馈控制。
- **全系大功率电机兼容**：一键开启真正的 `5kHz` 双路 PI 电流环（Id / Iq），支持传统低阻抗大功率航模电机与伺服电机。
- **人性化的“本圈相对位置”控制**：告别繁琐的绝对多圈角度计算，下发位置指令永远以当前机械圈的起点为基准，轻松实现指哪打哪。
- **极简高效的 SVPWM 调制**：摒弃传统繁琐的扇区判断，采用数学上完全等效的 **Min-Max 零序电压注入法 (马鞍波)**，用几行优雅的代码将直流母线电压利用率提升 15.47%，压榨电机极限性能。
- **VOFA+ 上位机实时调参监视**：内置 JustFloat 协议引擎，支持 100Hz 高速上报位置、速度、双路相电流以及 Iq 数据，支持串口实时解析下发指令。
- **大疆无人机启动音效**：利用高频无力矩的 Ud 电压注入，在不造成电机转动的情况下播放经典的 Do-Mi-So-Do 启动音效。

## 🛠️ 架构说明

系统采用严格的**双中断异步隔离架构**：
1. **5kHz 极速底层 (ADC 注入完成中断)**：处理电流采样、Clarke 变换、Park 变换、电流环 PI 控制、SVPWM / 正弦波换相。耗时极短，保证电流响应。
2. **1kHz 运动规划层 (TIM2 定时器中断)**：处理位置计算、速度差分、低通滤波、位置外环与速度外环 PID，以及处理串口的 VOFA+ 数据组包发送。

## 🚀 快速上手

### 1. 硬件连接
- **主控芯片**：STM32G431CBT6
- **电机驱动**：支持三相半桥驱动芯片（如 DRV8313, L6234 等）
- **编码器**：MT6701（SPI 模式）

### 2. 核心 API 示例

```c
#include "JerryFOC.h"

int main() {
    // ... STM32 HAL 初始化 ...

    // 配置是否启用电流内环 (0: 纯电压控制[适合云台电机], 1: 启用电流环[适合大功率电机])
    JerryFOC_useCurrentLoop(0); 

    // 播放启动提示音
    JerryFOC_playStartupSound();

    // 闭环前的开环标定 (寻找电气零点)
    JerryFOC_alignSensor();

    // 进入闭环速度控制模式
    JerryFOC_setMode(JERRYFOC_MODE_VELOCITY);
    JerryFOC_setVelocity(20.0f); // 目标速度 20 rad/s

    // 开启 1kHz 控制循环定时器
    HAL_TIM_Base_Start_IT(&htim2);

    while(1) {
        // 主循环处理换相 (极速响应 ADC 中断)
        JerryFOC_run();
    }
}
```

### 3. VOFA+ 串口通信调试
本代码通过 `USART1` 与上位机通信。支持的纯文本命令如下，可以直接通过 VOFA+ 发送：
- `Speed:20.0` -> 切换到速度模式并设定目标速度为 20 rad/s
- `Angle:3.14` -> 切换到位置模式并转到相对本圈半圈的位置 (PI)
- `Torque:0.5` -> 切换到扭矩模式并设定目标生力电流 (Iq) 为 0.5A

## 📚 核心函数 API 详解

| 函数名 | 作用与机制 |
|---|---|
| `JerryFOC_run()` | **底层核心心跳**：负责读取 ADC 注入组的相电流数据，执行 Clarke 与 Park 坐标变换，计算 PID 电流环/电压分配，并输出最终的 SVPWM/SPWM 占空比。它被放置在主循环中，依靠 ADC `JEOS` 标志位**严格按 5kHz 极速定频执行**。 |
| `JerryFOC_alignSensor()` | **开环标定**：上电时强制向特定电角度（通常是 $3\pi/2$ 或 0）注入固定电压，强行让转子对齐到该角度，借此记录磁编码器当前读数作为“电气零点”，是 FOC 换相能顺利进行的大前提。 |
| `JerryFOC_useCurrentLoop()` | **底层控制模式开关**：传入 `0` 时开启“纯电压前馈控制”，将外环计算出的目标直接作为电压下发，完全无视相电流底噪，是**云台电机**的唯一真神；传入 `1` 时则切入真正的带 PI 积分的电流闭环模式，是**大功率无刷/航模电机**的必备。 |
| `JerryFOC_setMode()` | **宏观状态机切换**：可无缝切换到 `JERRYFOC_MODE_TORQUE` (扭矩/电压)、`JERRYFOC_MODE_VELOCITY` (速度环) 或 `JERRYFOC_MODE_POSITION` (位置-速度串级环)。 |
| `JerryFOC_setPosition()` | **人性化相对位置控制**：它不采用死板的多圈绝对角度，而是以电机**当前所在的这一圈的 0 点为绝对基准**。例如电机已经转了 100 圈，你下发 3.14，它会直接转到第 100 圈的一半位置，免去了外部维护绝对圈数的麻烦。 |
| `JerryFOC_playStartupSound()` | **启动音效**：通过巧妙地在无效电角度（Id / D轴）注入极高频的交变电压，让电机线圈产生磁致伸缩发声，由于 Iq / Q轴没有力矩，所以电机只唱歌不乱转。 |

## 📈 VOFA+ 上位机配置与波形监视

为方便实时观测 PID 效果与电机底层状态，代码内置了 100Hz 高频上报。

**VOFA+ 软件设置：**
1. **协议选择**：在数据引擎选项中，选择 **`JustFloat`** 协议（底层代码发送的即为纯浮点数组加特殊帧尾）。
2. **控件选择**：使用 **波形图** 控件来观察数据的动态走向。

**7 大通道 (CH) 数据映射字典：**

- **CH0 (目标位置)**：用户下发或系统计算出的最终目标位置坐标 (rad)。
- **CH1 (实际位置)**：传感器反馈的实时多圈绝对机械角度 (rad)。
- **CH2 (目标速度)**：位置外环算出的目标速度，或是用户在纯速度模式下发的目标速度。
- **CH3 (实际速度)**：经过 `Tf=0.02s` 低通滤波器平滑处理后的电机真实转速 (rad/s)。
- **CH4 (实际 Iq)**：经过 Park 变换后的实时 Q轴（生力）电流。在开启电流环时，用于观察它是否平稳追踪目标电流；在电压模式时，用于观察负载施加的反抗电流。
- **CH5 (相电流 A)**：INA240 真实放大的 A 相电感原始电流 (A)。
- **CH6 (相电流 B)**：INA240 真实放大的 B 相电感原始电流 (A)。

*💡 **调试建议**：如果电机震动或发出刺耳啸叫，第一时间在 VOFA+ 勾选查看 `CH5` 和 `CH6`。如果它们不是完美的正弦波而是充满了锯齿，说明 ADC 采样受到 PWM 严重干扰或 `JerryFOC_useCurrentLoop(1)` 开错了。*
