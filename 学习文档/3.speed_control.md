# FOC 速度闭环控制与双环架构重构 (JerryFOC)

本篇文档记录了如何将 DengFOC 速度控制算法（V0.2）移植到我们的 STM32G431 工程中，并同时实现了一次极具工业水准的**“双环控制架构重组”**。

## 1. 为什么要做双环重构？
在之前纯搬运的 Arduino 代码中，所有的任务（读取编码器、PID 计算、FOC 换相）全都在一个没有时间概念的 `while(1)` 里面极速狂奔。这在简单的位置闭环下尚可工作，但遇到“速度闭环”时会遇到灾难性问题：
- **速度计算需要严格的时间差 ($\Delta T$)**。Arduino 代码里为了解决这个问题，在各个类的底部散落了大量的 `micros()` （微秒计时）函数。
- **微分噪声陷阱**：如果不受控地极速计算速度，$\Delta T$ 会趋近于 0，加上传感器底噪，会导致速度计算极度不稳定（前面已经探讨过）。

**重构后的 JerryFOC 双环架构：**
- **内环（电流/力矩换相环）**：在 `while(1)` 中通过 `JerryFOC_run()` **极速执行**，保证定子磁场死死咬住转子的任何微小运动，频率高达 PWM 更新率！
- **外环（速度 PID 环）**：放置在 `TIM2` 的定时器中断里，**严格按照 1ms 周期执行**。它只负责从容地计算速度、滤波、并输出目标力矩指令 `$U_q$` 给内环。这直接**消除了所有对系统时钟读取 `micros()` 的依赖**！

## 2. 核心代码解析对照

### 2.1 低通滤波 (Low Pass Filter) 的移植
速度反馈信号含有高频噪声，必须使用低通滤波。
**原版 DengFOC C++ 代码 (需动态读取 `micros()`)：**
```cpp
float LowPassFilter::operator() (float x) {
    unsigned long timestamp = micros();
    float dt = (timestamp - timestamp_prev)*1e-6f;
    float alpha = Tf/(Tf + dt);
    float y = alpha*y_prev + (1.0f - alpha)*x;
    // ...
```
**JerryFOC C 代码 (TIM2 定时器加持版)：**
因为我们在 1ms 定时器中断里调用，`dt` 是绝对固定的 `0.001s`，这不仅让算法瞬间变得极度优雅，而且极大节省了浮点除法的运算资源！
```c
float JerryFOC_LPF_Calc(LowPassFilter* filter, float x) {
    float Ts = 0.001f; // 固定的 1ms 完美步长
    // 计算权重：常数 / (常数 + 步长)
    float alpha = filter->Tf / (filter->Tf + Ts);
    // 新平滑值 = 权重 * 老平滑值 + (1-权重) * 最新粗糙值
    float y = alpha * filter->y_prev + (1.0f - alpha) * x;
    filter->y_prev = y;
    return y;
}
```

### 2.2 PID 控制器的移植
和滤波同理，我们将原本带有复杂防跳跃处理的动态步长 PID，改成了极致高效的定步长 C 语言算法。
**JerryFOC C 代码：**
```c
float JerryFOC_PID_Calc(PIDController* pid, float error) {
    float Ts = 0.001f; // 固定的 1ms 步长
    
    // P 环：当前误差直出
    float proportional = pid->P * error;
    
    // I 环：梯形面积积分法 (Tustin)
    float integral = pid->integral_prev + pid->I * Ts * 0.5f * (error + pid->error_prev);
    integral = _constrain(integral, -pid->limit, pid->limit); // 严防积分饱和爆表
    
    // D 环：误差的变化率
    float derivative = pid->D * (error - pid->error_prev) / Ts;

    float output = proportional + integral + derivative;
    output = _constrain(output, -pid->limit, pid->limit);

    // 变化率限幅 (Ramp): 防止电压瞬间跃变撕裂电机
    if(pid->output_ramp > 0){
        float output_rate = (output - pid->output_prev) / Ts;
        if (output_rate > pid->output_ramp) output = pid->output_prev + pid->output_ramp * Ts;
        else if (output_rate < -pid->output_ramp) output = pid->output_prev - pid->output_ramp * Ts;
    }
    
    pid->integral_prev = integral;
    pid->output_prev = output;
    pid->error_prev = error;
    return output;
}
```

### 2.3 外环任务的部署 (`TIM2` 定时器中断)
在 `JerryFOC.c` 底部，我们使用了 HAL 库标准的 `HAL_TIM_PeriodElapsedCallback` 来接管外环：
```c
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim) {
    if (htim->Instance == TIM2) {
        // 1. 获取包含多圈累加的精确角度
        static float last_mechanical_angle = 0;
        float current_mechanical_angle = JerryFOC_getAngle(); 
        
        // 2. 算裸速度 (1ms走过的弧度 / 0.001)
        float raw_velocity = (current_mechanical_angle - last_mechanical_angle) / 0.001f;
        last_mechanical_angle = current_mechanical_angle;
        
        // 3. 过滤噪音，得到丝滑的速度反馈
        filtered_velocity_global = JerryFOC_LPF_Calc(&M0_Vel_Flt, DIR * raw_velocity);
        
        // 4. 算出速度闭环误差
        float error = JerryFOC_Target_Velocity - filtered_velocity_global;
        
        // 5. 将误差丢给速度 PID，计算得出内环急需的推力（电压 Uq），存入全局变量！
        JerryFOC_Target_Uq = JerryFOC_PID_Calc(&vel_loop_M0, error);
    }
}
```

### 2.4 USART1 串口指令接收
我们在 `main.c` 的前部通过 `HAL_UART_Receive_IT` 配置了逐字节中断接收。当检测到换行符 `\n` 时，立刻用 `atof` 将字符解析为浮点数，并下发给速度环！
这取代了 Arduino 原版中繁杂的 `serialReceiveUserCommand`。

## 3. 运行指南与联调
现在，你的代码架构极度干净。启动板子后：
1. **零点标定**：电机会锁死 3 秒。
2. **连接串口助手**：波特率选择 115200（需跟你的 CubeMX 配置一致）。
3. **发送指令**：
   - 输入 `10.0` 并加回车发送，电机将平滑加速至 10 rad/s。
   - 输入 `0.0`，电机会立刻刹车至静止（这是 PID 正在输出阻力，并非断电闲置）。
   
享受属于你自己的专业伺服控制系统吧！
