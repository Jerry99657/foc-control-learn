# FOC 闭环位置控制算法移植指南 (Arduino -> STM32 HAL)

本文档旨在介绍如何将邓哥 FOC（基于 Arduino）的**闭环位置控制算法**，结合我们上一节移植好的 `MT6701` 磁编码器驱动，无缝移植到本项目的 STM32G431 平台中。

## 1. 移植综述
闭环控制与开环控制的最大区别在于：**我们需要真实反馈**。
- **开环**：我们“盲目”地随时间积分角度，并强行输出该角度的磁场拖拽转子。
- **闭环位置**：我们通过 `MT6701` 读取电机转子当前真实的机械角度，利用 **PID（此处仅用 P 比例控制器）** 计算当前角度与目标角度的误差，根据误差输出相应的 $U_q$（力矩），从而精准驱动电机到达并保持在目标位置。

核心新增了两个关键步骤：
1. **传感器零点对齐 (`alignSensor`)**：上电时强制定子产生一个固定方向的磁场，吸住转子，从而校准机械角度与电角度的绝对零点关系。
2. **P 闭环控制器 (`positionCloseLoop`)**：取代了时间积分，实时根据位置偏差计算输出电压。

## 2. 代码移植对照与解析

### 2.1 零点校准函数 (`alignSensor`)
在原本的 Arduino 代码中，通过强制输出 `setPhaseVoltage(3, 0, _3PI_2)` 并延时 3 秒，让转子锁定，随后读取此时的绝对角度作为零点偏移。
**STM32 移植版本：**
```c
// 传感器对齐与电角度零点标定
void alignSensor(void) {
    // 强制输出 Uq=3V, Ud=0V, 电角度=270度(3PI/2)，让电机强行对齐到磁场绝对零点
    setPhaseVoltage_CL(3.0f, 0.0f, 3.0f * PI / 2.0f);
    
    // 阻塞等待3秒，确保电机完全停止震荡并锁死在原点
    HAL_Delay(3000); 
    
    // 读取此时的 MT6701 机械角度 (Angle)，乘以极对数与方向，记录为零点偏移
    cl_zero_electric_angle = (DIR * POLE_PAIRS) * Angle;
    
    // 初始化多圈跟踪变量，并将此时的物理角度定义为目标 0 度
    prev_Angle = Angle;
    full_rotations = 0;
    mechanical_zero_offset = Angle;
    
    // 标定完成，关闭输出
    setPhaseVoltage_CL(0.0f, 0.0f, 3.0f * PI / 2.0f);
}
```
*注：这里的 `Angle` 是我们在 `MT6701.c` 中通过 SPI 读取并更新的全局变量。*

### 2.2 电角度实时计算 (`_electricalAngle_CL`)
开环中电角度是随时间积分的，闭环中电角度完全由传感器真实回传。
**STM32 移植版本：**
```c
float _electricalAngle_CL(void) {
    // (实际机械角度 * 方向 * 极对数) - 标定好的零点偏移 = 实际电角度
    return _normalizeAngle_CL((DIR * POLE_PAIRS) * Angle - cl_zero_electric_angle);
}
```

### 2.3 闭环位置控制核心 (`positionCloseLoop`)
原本的 Arduino 代码：
```c
setPhaseVoltage(_constrain(Kp*(motor_target-DIR*Sensor_Angle)*180/PI,-6,6),0,_electricalAngle());
```
**STM32 移植版本：**
```c
// 闭环位置控制函数
// target_angle_rad: 用户期望电机到达的目标机械角度 (弧度)
void positionCloseLoop(float target_angle_rad) {
    float current_Angle = Angle;
    float d_angle = current_Angle - prev_Angle;
    
    // 跨越 2PI 的跳变检测
    if(d_angle > PI) {
        full_rotations -= 1.0f; 
    } else if(d_angle < -PI) {
        full_rotations += 1.0f; 
    }
    prev_Angle = current_Angle;
    
    // 计算包含多圈的真实机械角度，并减去标定时的零点位置，确保 target=0 刚好在标定点
    float Sensor_Angle = full_rotations * 2 * PI + current_Angle - mechanical_zero_offset; 
    
    float Kp = 0.133f; // P控制器比例系数
    
    // 计算位置误差，输出目标 Uq 电压（交轴电压，即力矩）
    float Uq = Kp * (target_angle_rad - DIR * Sensor_Angle) * 180.0f / PI;
    
    // 电压限幅，防止烧毁电机或过调制 (12V供电限幅6V)
    Uq = _constrain(Uq, -6.0f, 6.0f); 
    
    // 直接根据传感器反馈的实时电角度，生成对应的三相磁场
    setPhaseVoltage_CL(Uq, 0.0f, _electricalAngle_CL());
}
```

## 3. 如何在工程中运行 (main.c)

你现在已经拥有了 `MT6701` 驱动和 `close_loop` 的算法代码。在 `main.c` 中，你应该这样组织：

```c
/* USER CODE BEGIN Includes */
#include "MT6701.h"
#include "close_loop.h"
/* USER CODE END Includes */

int main(void)
{
  // ... 系统初始化代码 ...
  
  /* USER CODE BEGIN 2 */
  // 1. 开启 PWM 输出
  HAL_TIM_PWM_Start(&htim1, TIM_CHANNEL_1);
  HAL_TIM_PWM_Start(&htim1, TIM_CHANNEL_2);
  HAL_TIM_PWM_Start(&htim1, TIM_CHANNEL_3);
  
  // 2. 触发第一次 MT6701 的 SPI DMA 读取，建立后台循环更新
  HAL_SPI_TransmitReceive_DMA(&hspi1, MT6701_Data, MT6701_Data, 3);
  
  // 3. 执行电机零点对齐标定 (注意电机在上电时会强制转动一下然后锁死3秒)
  alignSensor();
  /* USER CODE END 2 */

  /* Infinite loop */
  /* USER CODE BEGIN WHILE */
  while (1)
  {
    // 调用闭环位置控制，目标角度设为 3.14 (180度半圈)
    positionCloseLoop(3.14f);
    
    // 可以添加较小的延迟防止CPU占满，由于是完全闭环反馈，循环速度不再受极度严格的约束
    // HAL_Delay(1); 
    /* USER CODE END WHILE */
  }
}
```

## 4. 总结
**闭环位置算法的核心哲学就是**：不管外部有什么干扰，只要我的真实角度不到目标位置，我就根据误差产生相应的电压 $U_q$ 来推拉转子。当它到达目标位置时，误差为 0，输出的力矩也降为 0，完美悬停。这也就是伺服电机的基本工作原理！
