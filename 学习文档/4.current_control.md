# FOC 第四步：电流环移植指南 (基于 DengFOC v0.3)

本报告详细说明如何将 DengFOC v0.3 中的电流环控制逻辑无缝移植到你自己的 `JerryFOC` 库中。

## 1. DengFOC 电流环核心逻辑解析

在 DengFOC v0.3 中，电流环的控制逻辑主要分为三个阶段：
1. **电流采样与坐标变换 (Clarke & Park)**：通过 `InlineCurrent.cpp` 读取相电流，然后在 `cal_Iq_Id` 中进行 Clarke 和 Park 变换提取 $I_q$。
2. **电流低通滤波**：对高频跳变的 $I_q$ 信号使用一阶低通滤波处理。
3. **PID 计算与电压输出**：在 `DFOC_M0_setTorque` 中，将目标电流与滤波后的 $I_q$ 进行 PID 运算，输出修正后的目标电压 $U_q$。

---

## 2. 代码对应关系与移植指南

### 第一部分：相电流采样与 Clarke/Park 变换

**DengFOC (Arduino) 实现：**
```cpp
// 通过Ia,Ib,Ic计算Iq,Id(目前仅输出Iq)
float cal_Iq_Id(float current_a,float current_b,float angle_el)
{
  // 1. Clarke 变换 (等面积/恒幅值变换)
  float I_alpha=current_a;
  float I_beta = _1_SQRT3 * current_a + _2_SQRT3 * current_b;

  // 2. Park 变换
  float ct = cos(angle_el);
  float st = sin(angle_el);
  float I_q = I_beta * ct - I_alpha * st; // 仅计算 q 轴电流
  return I_q;
}
```

**移植到 JerryFOC (STM32) 的编写方法：**
在 STM32 中，由于算力充足，我们可以在底层实现标准的 Clarke 和 Park 函数，完全复现 DengFOC 的逻辑：
```c
// 1. Clarke 变换 (移植 DengFOC 逻辑)
void JerryFOC_Clarke(float Ia, float Ib, float *Ialpha, float *Ibeta) {
    *Ialpha = Ia;
    *Ibeta = (1.0f / sqrtf(3.0f)) * Ia + (2.0f / sqrtf(3.0f)) * Ib;
}

// 2. Park 变换 (严格对应 DengFOC 的符号公式)
float JerryFOC_Park(float Ialpha, float Ibeta, float angle_el) {
    float ct = cosf(angle_el);
    float st = sinf(angle_el);
    return Ibeta * ct - Ialpha * st;
}
```

### 第二部分：电流获取与低通滤波

**DengFOC (Arduino) 实现：**
```cpp
float DFOC_M0_Current()
{  
  // 1. 获取原始 Iq
  float I_q_M0_ori=cal_Iq_Id(CS_M0.current_a,CS_M0.current_b,_electricalAngle());
  // 2. 使用 Tf=5ms 的低通滤波器进行滤波
  float I_q_M0_flit=M0_Curr_Flt(I_q_M0_ori);
  return I_q_M0_flit;  
}
```

**移植到 JerryFOC (STM32) 的编写方法：**
在 Arduino 中这个函数是轮询调用的，而在 STM32 中，我们需要在最高优先级的 5kHz 循环（如 TIM1/ADC注入中断引发的 `JerryFOC_run()`）中定步长处理：
```c
// 在 JerryFOC_run() 中处理
float Ialpha, Ibeta;
JerryFOC_Clarke(phase_current_a, phase_current_b, &Ialpha, &Ibeta);
float raw_Iq = JerryFOC_Park(Ialpha, Ibeta, electrical_angle);

// 低通滤波 (移植 DengFOC 的低通滤波算法)
// M0_Curr_Flt 需要在初始化时定义好，Tf 根据系统噪声设置 (如 0.005f)
iq_current = JerryFOC_LPF_Calc(&M0_Curr_Flt, raw_Iq);
```

### 第三部分：电流环 PID 与电压输出

**DengFOC (Arduino) 实现：**
```cpp
void DFOC_M0_setTorque(float Target)
{
  // current_loop_M0 是一个 PID 控制器类
  // 传入误差 (Target - 实际Iq)，输出修正电压，然后传给 setTorque 进行反 Park 和反 Clarke 变换
  setTorque(current_loop_M0(Target-DFOC_M0_Current()),_electricalAngle());
}
```

**移植到 JerryFOC (STM32) 的编写方法：**
由于 STM32 采用 C 语言，你需要展开 PID 类的计算过程：
```c
float Uq = 0.0f;
// 计算电流 PID 误差 (目标电流 - 滤波后的实际电流)
float error = Target_Iq - iq_current;

// 比例项计算
float proportional = curr_loop_M0.P * error;

// 积分项计算 (采用定步长矩形积分)
float integral = curr_loop_M0.integral_prev + curr_loop_M0.I * Ts * error;
// 积分限幅防风饱和
integral = _constrain(integral, -curr_loop_M0.limit, curr_loop_M0.limit); 
curr_loop_M0.integral_prev = integral;

// 计算总输出交轴电压 Uq
Uq = proportional + integral;
Uq = _constrain(Uq, -curr_loop_M0.limit, curr_loop_M0.limit);

// 执行换相：这里的 Uq 等效于 DengFOC 传入 setTorque 的第一个参数
JerryFOC_setPhaseVoltage(Uq, 0.0f, electrical_angle);
```

## 3. 级联控制架构：从 DengFOC 到 JerryFOC

在 DengFOC 中，如果你想要实现“速度-电流”双闭环，代码往往是将各种控制接口堆叠。而在你的 `JerryFOC` 库中，推荐使用更为规范的**双环级联定时器架构**：

1. **外环：1kHz 定时器（TIM2）**
   - 读取编码器角度，计算物理速度。
   - 对物理速度进行低通滤波。
   - 运行速度外环 PID，计算出需要的输出扭矩。
   - 将输出扭矩幅值赋给全局变量 `JerryFOC_Target_Iq`。

2. **内环：5kHz 甚至 10kHz 的极速循环（JerryFOC_run）**
   - 实时采集 ADC 相电流。
   - 完成 Park 和 Clarke 变换得到 $I_q$。
   - 利用 `JerryFOC_Target_Iq` 与实际 $I_q$ 运行电流内环 PID。
   - 最终输出 $U_q$ 并执行 FOC 换相。

> [!CAUTION]
> **移植避坑指南 (重要！)**
> 虽然上述代码是 DengFOC 的原版电流环逻辑，但 DengFOC v0.3 的电流环主要针对的是航模级大功率低内阻电机。
> 如果你在高内阻、极低电感的小型云台电机上强行引入这套电流环，**务必将电流环的 P 参数降到极低**（例如 `P=0.01` 到 `0.05`），否则极低的相电感会导致严重的 5kHz 极限环高频震荡。
> 针对云台电机，如果电流环无法调稳，行业内推荐的标准降级方案是直接**旁路电流环 (Voltage Mode)**。
